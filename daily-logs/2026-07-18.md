## What I Learned

- Terraform state maps configuration addresses to real infrastructure and must be protected.
- Import connects an existing resource to Terraform state but does not automatically generate or validate matching configuration.
- Recreating resources is safest first in an isolated dev environment.
- Long-lived infrastructure belongs in Terraform, while website uploads and CloudFront invalidations belong in GitHub Actions.
- A Terraform plan must be read literally: the Day 10 plan contained outputs only and proposed no AWS infrastructure changes.
- `.terraform.lock.hcl` supports reproducible provider selection and should be committed.
