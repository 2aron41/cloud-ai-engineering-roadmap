## What I Learned

- `terraform init` prepares the working directory and installs providers.
- `terraform fmt` standardizes configuration formatting.
- `terraform validate` checks configuration consistency.
- `terraform plan` previews proposed infrastructure actions.
- Providers translate Terraform configuration into API operations.
- `.terraform.lock.hcl` should be committed, while `.terraform/` should not.
- State files may contain sensitive data and must be protected.
- Variable validation can reject unsafe inputs before infrastructure changes.
